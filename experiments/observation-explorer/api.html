GeoTIFF Catalog API Guide

The Sentinel-2 GeoTIFF Timeline Explorer loads catalog entries from a simple JSON API. This guide documents the required payload so that your service (or pasted response) matches the viewer’s expectations.

Endpoint

Method: GET
Content-Type: application/json
Response: JSON array of catalog entries

Request example

GET https://api.example.com/geotiff/catalog
Accept: application/json

Response schema

Each response item represents one GeoTIFF product.

Field Type Required Description
name string No Human-friendly label used in the timeline list.
date string (ISO 8601) Yes Acquisition date. Use YYYY-MM-DD or an ISO 8601 timestamp such as 2024-06-02T00:00:00Z.
cloud number No Cloud fraction between 0 and 1. The UI converts this to a percentage.
url string (URL) Yes Direct link to the Cloud Optimized GeoTIFF (COG).

Response example

[
  {
    "name": "Spring 2024",
    "date": "2024-06-02",
    "cloud": 0.05,
    "url": "https://cdn.example.com/cogs/sentinel-2/2024-06-02.tif"
  },
  {
    "name": "Autumn 2024",
    "date": "2024-10-18",
    "cloud": 0.18,
    "url": "https://cdn.example.com/cogs/sentinel-2/2024-10-18.tif"
  }
]

Best practices

  • Return entries sorted by date (ascending) for smoother browsing.
  • Ensure the API sends CORS headers (for example, Access-Control-Allow-Origin: *) so the browser can fetch your catalog.
  • Use HTTPS URLs for GeoTIFF assets to avoid mixed-content issues.

Error handling

Return standard HTTP error codes. The viewer surfaces 4xx/5xx responses as load failures.
